## 1. How env vars actually load (the mental model)

- `fastworkflow.init(env_vars=dict)` stores the dict in a module-global (`fastworkflow/__init__.py`).
  The CLI builds that dict as `{**dotenv_values(env_file), **dotenv_values(passwords_file)}` —
  passwords merge LAST, so key material wins over anything in the env file.
- **Precedence quirk (verified in `fastworkflow/__init__.py` `get_env_var`)**: if a caller
  supplies a code default and the var is absent from the init dict, the default is returned
  WITHOUT consulting `os.environ`. Shell `export` therefore cannot override defaulted vars
  such as `SESSION_STATE_STORE`, `INTENT_DETECTION_TINY_MODEL`,
  `INTENT_DETECTION_LARGE_MODEL`. Put overrides in the env FILE.
- `fastworkflow build` and `fastworkflow refine` call `fastworkflow.init(env_vars={})`
  (build/__main__.py, refine/__main__.py) — they load **no env files**. Their LLM config
  (`LLM_COMMAND_METADATA_GEN`, `LITELLM_API_KEY_COMMANDMETADATA_GEN` — note the missing
  underscore in `COMMANDMETADATA`) must be OS-exported. Neither var appears in the bundled
  template; they live only in the repo-local `env/.env` / `passwords/.env`.
- Import side-effect (litellm's, not fastworkflow's): litellm runs `load_dotenv()` over
  the CWD `.env` at import time (`site-packages/litellm/__init__.py`). From the repo root
  that `.env` is the gen-env merge, which can contain the template's placeholder keys —
  verified 2026-07-09: importing fastworkflow from the repo root added 7
  `LITELLM_API_KEY_*` vars to `os.environ` (6 with `<...>` placeholder values), while
  importing from a directory with no `.env` added none. This is why placeholder detection
  (`<` or `your-` in the value) exists in
  `tests/test_train_modern_stack.py::_looks_like_real_key`.

## 3. gen-env.sh, line by line (verified 2026-07-09)

- Inputs: `find . -name "*.env"` excluding `./.env` itself, `./override/*`, and any
  `--exclude dir1,dir2` paths. **Everything else matching `*.env` anywhere in the tree is
  merged** — including `env/.env`, `passwords/.env`, `passwords/pypi.env`, the bundled
  example templates, and any fetched `./examples/*.env`.
- Then files under `override/` (if the directory exists) are processed last → override
  semantics. No `override/` directory exists in the repo today; it is an opt-in
  per-developer convention (undocumented elsewhere — this paragraph is the documentation).
- Last occurrence of a key wins; output is sorted; written to root `.env` with a
  do-not-edit header.
- Invoked by `make gen-env` and (transitively) `make publish-testpypi` / `make publish`.

Foot-guns, ranked:

1. **Tracked-file leak**: keys pasted into `env/.env` (or any new tracked `*.env`) get
   committed AND merged. Keys belong only in `passwords/` (ignored) or untracked files.
2. **Placeholder clobbering**: the bundled template's `<API KEY ...>` values participate
   in the merge. They sort/merge like any value; whether your real key survives depends on
   find's directory order. If root `.env` ends up with a `<...>` value after `make gen-env`,
   your real key lost the merge — check for duplicate key names across `*.env` files.
3. **Secret aggregation**: root `.env` concentrates every secret in one plaintext file at
   the repo root. Never `cat` it in a shared terminal, CI log, or LLM conversation.

## 4. Per-workflow files at run/train time

`fastworkflow train|run|run_fastapi_mcp <workflow> [env] [passwords]` default the two
optional paths to `<workflow>/fastworkflow.env` and `<workflow>/fastworkflow.passwords.env`
(`cli.py::find_default_env_files` — the `--help` text claiming ".env in current directory"
is wrong). For fetched examples, the env files live at `./examples/*.env` (one level UP
from the workflow), so you must pass both paths explicitly; the CLI error message tells
you the exact command if you forget.

Required-var floor (train and run hard-fail without it): `SPEEDDICT_FOLDERNAME`
(conventionally `___workflow_contexts`) — it doubles as the "did the env file load at all"
probe. `run` additionally hard-fails if `LITELLM_API_KEY_SYNDATA_GEN` is absent
(run/__main__.py:129-132), even though run never generates synthetic data — keep the key
set. The parameter-extraction sentinels (`MISSING_INFORMATION_ERRMSG`,
`INVALID_INFORMATION_ERRMSG`, `NOT_FOUND`, `INVALID`, `PARAMETER_EXTRACTION_ERROR_MSG`)
have no code defaults and must be present in the env file (the templates carry them).

## 5. Role/key matrix

| Model var (env file) | Key var (passwords file) | Consumed by |
|---|---|---|
| `LLM_SYNDATA_GEN` | `LITELLM_API_KEY_SYNDATA_GEN` | train: synthetic utterances + DSPy few-shot example generation |
| `LLM_PARAM_EXTRACTION` | `LITELLM_API_KEY_PARAM_EXTRACTION` | run: parameter extraction (DSPy) |
| `LLM_AGENT` | `LITELLM_API_KEY_AGENT` | run: the agent loop |
| `LLM_PLANNER` | `LITELLM_API_KEY_PLANNER` | run: agent planner |
| `LLM_CONVERSATION_STORE` | `LITELLM_API_KEY_CONVERSATION_STORE` | run_fastapi_mcp: conversation topic/summary generation |
| `LLM_COMMAND_METADATA_GEN` | `LITELLM_API_KEY_COMMANDMETADATA_GEN` | build/refine GenAI post-processing (OS env only; not in templates) |
| `LLM_RESPONSE_GEN` | `LITELLM_API_KEY_RESPONSE_GEN` | **Dead config** — zero consumers in `fastworkflow/` or `tests/` (verified by ripgrep 2026-07-09); appears only in templates/docs. Harmless to fill, pointless to debug. |

LiteLLM proxy mode: prefix any model value with `litellm_proxy/` → `LITELLM_PROXY_API_BASE`
becomes mandatory (ValueError if unset), `LITELLM_PROXY_API_KEY` optional, and the
per-role key is ignored for that call (`fastworkflow/utils/dspy_utils.py::get_lm`).

## 6. Re-verification commands

```sh
git ls-files env/.env                        # expect: env/.env  (tracked!)
git check-ignore -v .env passwords/.env      # expect: both ignored
grep -oE '^[A-Z_]+' passwords/.env           # key NAMES only — never print values
sed -n '85,120p' gen-env.sh                  # merge/override/output logic
grep -rn "LLM_RESPONSE_GEN" fastworkflow/ tests/ --include='*.py'   # expect: no consumers
grep -n 'init(env_vars={})' fastworkflow/build/__main__.py fastworkflow/refine/__main__.py
grep -n -A5 'def find_default_env_files' fastworkflow/cli.py
```
